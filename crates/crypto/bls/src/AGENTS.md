# src

## Purpose
BLS signature scheme implementations for both BLS12-377 and BLS12-381 curves using the W3F (Web3 Foundation) `tnt-bls` library. Provides key generation, signing, verification, and signature aggregation conforming to the `blueprint-crypto-core` `KeyType` and `AggregatableSignature` traits.

## Contents (one hop)
### Subdirectories
- (none)

### Files
- `lib.rs` - Crate root. Helper `to_bytes`/`from_bytes` functions for ark-serialize round-tripping. `CONTEXT` constant (`b"tangle"`) used as the signing domain separator. `impl_w3f_serde!` macro generating newtype wrappers with `BytesEncoding`, `Serialize`/`Deserialize`, `Eq`/`Ord`/`Hash` for public keys, secret keys, and signatures. `define_bls_key!` macro generating full `KeyType` implementations for each curve variant. Invoked as `define_bls_key!(Bls377, Bls381)` to produce `bls377` and `bls381` submodules.
- `aggregation.rs` - `AggregatableSignature` impls for `W3fBls377` and `W3fBls381`. Aggregates by summing public key group elements and signature group elements. `verify_aggregate` delegates to single-message `KeyType::verify`.
- `error.rs` - `BlsError` enum with variants: `InvalidSeed`, `HexError`, `InvalidSignature`, `InvalidInput`. Type alias `Result<T>`.
- `tests.rs` - Tests for both BLS377 and BLS381: key generation determinism, sign/verify, pre-hashed signing, serialization round-trips, aggregation success/failure, mismatched keys, empty aggregation. Uses `impl_crypto_tests!` macro from `blueprint-crypto-core`.

## Key APIs
- `bls377::W3fBls377` / `bls381::W3fBls381` -- `KeyType` implementations (generated by `define_bls_key!` macro)
- `W3fBls377Public`, `W3fBls377Secret`, `W3fBls377Signature` (and Bls381 equivalents) -- newtype wrappers
- `AggregatableSignature` impls for both curve variants -- `aggregate()`, `verify_aggregate()`
- `to_bytes()` / `from_bytes()` -- ark-serialize helpers

## Relationships
- Depends on `blueprint-crypto-core` for `KeyType`, `KeyTypeId`, `BytesEncoding`, `AggregatableSignature` traits
- Depends on `tnt-bls` for the underlying BLS curve operations
- Depends on `blueprint-crypto-hashing` (in tests) for `sha2_256`
- Sibling to `blueprint-crypto-bn254`, `blueprint-crypto-ed25519`, `blueprint-crypto-k256`, `blueprint-crypto-sr25519`
- Re-exported through `blueprint-crypto` meta-crate
